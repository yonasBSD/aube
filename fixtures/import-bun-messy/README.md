Messy Bun workspace fixture for import/install coverage.

This fixture is intentionally broader than the first BATS smoke test will
need. Normal registry packages resolve from the test Verdaccio registry, but
the remote tarball and git entries still hit the external internet.

Regenerate `bun.lock` with `./regenerate-lock.sh` after starting the test
Verdaccio registry.

Feature coverage:

- Bun text `bun.lock` generated by a real `bun install`.
- `bunfig.toml` with `install.linker = "isolated"`.
- Root workspace config with `packages/*` importers.
- Four workspace package importers: `@fixture/core`, `@fixture/plugin`,
  `@fixture/legacy`, and `@fixture/range-user`.
- Workspace dependencies using `workspace:*` and `workspace:^`.
- Nested workspace-to-workspace dependencies and a workspace peer dependency.
- A workspace package present only as a workspace importer, not as a root
  dependency.
- Default catalog dependency via `catalog:`.
- Named catalog dependencies via `catalog:colors`, `catalog:evens`, and
  `catalog:utils`.
- Mixed catalog and non-catalog dependencies across root and workspace
  importers.
- Top-level `overrides` for `is-number@<7.0.0`.
- Top-level `patchedDependencies` for `is-odd@3.0.1`.
- Top-level `trustedDependencies` containing `aube-test-builds-marker`.
- Trusted build-script package coverage through `aube-test-builds-marker`.
- Untrusted build-script package coverage through
  `aube-test-builds-marker-2`.
- Registry exact pins such as `aube-test-builds-marker@1.0.0` and
  `aube-test-builds-marker-2@1.0.0`.
- Registry ranges such as `is-number@^6.0.0`, `is-number@^7.0.0`, and
  `kind-of@^3.2.2`.
- npm alias dependencies such as `npm:is-odd@3.0.1`.
- Remote registry tarball dependency for `is-odd@3.0.1`.
- Local folder dependency via `file:./vendor/local-dir`.
- Local folder package with its own nested dependency on `is-number@^6.0.0`.
- Local tarball dependency via `file:./vendor/local-tar-1.0.0.tgz`.
- Local tarball package generated from `vendor/local-tar-src/package`.
- Remote git dependency using bare GitHub shorthand, `github:tj/term-canvas`.
- Remote git dependency using GitHub shorthand plus shorthash.
- Remote git dependency using GitHub shorthand plus long hash.
- Remote git dependency using `git+https://` plus long hash.
- Multiple workspace importers requesting the same package with different
  versions and specifier formats.
- Workspace packages using both exact and range registry specifiers.
- Root and nested importers both referencing the same local folder and local
  tarball packages.

Known intentional gaps:

- `bunfig.toml` includes Bun's `minimumReleaseAge` settings for fixture
  realism, but aube does not read Bun's age gate from `bunfig.toml` yet.
- Git entries use the public `tj/term-canvas` repo for URL specifier shapes.
  That repo is intentionally old and unlikely to change, which keeps the
  remote-git coverage low-variance while still exercising real GitHub specs.
  Tests that need fully hermetic execution should replace these with local
  `git+file://` repos.
- The remote tarball dependency intentionally points at npmjs.org instead of
  Verdaccio so the fixture keeps one real remote tarball URL shape.
